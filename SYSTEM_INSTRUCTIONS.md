# OpenClaw システム憲法 (SYSTEM_INSTRUCTIONS)

## 1. プロジェクト目的
日本の不動産価格データを元に、AIが記事・分析・ツールページを自動生成し、継続的に改善される不動産データサイトを構築・運用する。

## 2. システム構成・フロー
以下のAIスキル・スクリプトを連携させ、自律的なループを実行する。

1.  **データ取得**: `api_fetcher.py` (国交省API)
2.  **価格分析**: `price_analyzer.py` / `calculators.py`
3.  **記事・画像生成**: `article_generator.py` / `visual_generator.py`
4.  **最適化**: `llmo_optimizer.py` (AI検索向け) / `content_optimizer.py` (アクセス統計ベース)
5.  **内部リンク**: `internal_link_builder.py`
6.  **サイト構築・公開**: `deploy_site.py` / `publish_site.py` (Hugo + GitHub Pages)
7.  **自律ループ**: `orchestrator.py` による自動サイクル（公開 → 分析 → 改善 → 再公開）

## 3. ページタイプ
- **エリアページ**: 市区町村ごとの価格推移・分析。
- **住所ページ**: 町名単位の成約データ・詳細分析。
- **ツールページ**: 利回り、ローン計算、家賃相場、価格予測。
- **AI分析ページ**: 将来予測、投資評価。

## 4. UI/UX & デザインルール
- **ミニマルデザイン**: 不要な装飾を排し、情報を整理する。
- **スマホ優先**: モバイルでの閲覧性を最優先する。
- **高速表示**: 静的サイト生成 (Hugo) を活用し、軽量なアセットのみを使用。
- **クリーンな画像**: グラフやサムネイルに「OpenClaw」「AI Generated」「Tool Name」などの文字を一切入れない。

## 5. SEO / LLMO 戦略
AI検索（ChatGPT, Perplexity, Google AI Overviewsなど）に引用されやすい構造を維持する。
- 記事冒頭に **「要約」** を作成。
- 重要なデータは **「表」** で整理。
- **「FAQ」** セクションを設ける。
- 検索意図に沿った **「見出し最適化」** を行う。

## 6. 自動改善ループ
オーケストレーターは定期的に `usage_stats.json` とアクセスログを確認し、以下のサイクルを回す。
`公開` → `アクセス分析` → `人気トピック発見` → `コンテンツ改善` → `記事追加`
